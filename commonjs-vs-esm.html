什麼是 CJS ? 什麼是 ESM ?

milletbard
2022-06-16

[toc]

CommonJS(CJS) scripts 規範已被廣泛使用在 Node.js 的伺服器端開發，它也用於瀏覽器端的開發，但因為瀏覽器端不支援 CommonJS ，所以程式碼必須經過編譯器轉譯，而 ESM scripts (ES6 aka ES2015)另一個主流模組。

CommonJS 可以通過使用 require() 和 module.exports 來使用，而 ESM 則是使用 import 和 export 語法來實現類似 (其實不太一樣) 的功能。

ESM 和 CJS 是完全不同的。儘管從表面上來看，ESM 看起來非常相似。

什麼是 CJS ? 什麼是 ESM ?

從 Node 誕生開始，Node Module 就透過 CommonJS 進行開發，並透過 require() 引入它們，當實作了 script，提供給他人使用，可以定義 exports,使用 命名導出 module.exports.foo = 'bar' 或是 默認導出 module.exports = 'baz' 的方式導出 script。

ESM 和 CJS 是完全不同的

CJS 是 require 是同步的，它不會回傳 Promise 或 callback 。

ESM ，模組加載器 (Module Loader) 會以非同步的方式加載。在第一階段， ESM Loader 會在不運行程式碼的情況下解析 import 與 export；在解析階段， ESM Loader 可以在沒有真正 運行程式碼之前 立即的找出 命名導入 因拼寫錯誤而引發的異常。

之後 ESM Module Loader 會下載以及解析我們導入的所有 scripts，再來會輪到 scripts 所依賴的 scripts，然後建構依賴的 模組地圖 (ES module graph)，直到找到沒有再導入任何 scripts 的 script。最後，允許執行該 scripts，再允許運行依賴該 script 的 script，以此類推。

模組地圖 (ES module graph)的所有 兄弟 (sibling) 的 script 都是併行下載的，但它們會按照順序執行，由 Module Loader 的規範保證。

ESM 改變了很多事情

ESM 改變了 Javascript 很多事情，首先，ESM script 默認使用嚴格模式 (use strict)，它們的 this 不會引用全局物件，作用域的運作方式不同等等。

這就是為什麼即使在瀏覽器中，<script> 標籤的默認設定不是 ESM 的；必須添加 type="module" 才可以進入 ESM 模式，將默認設定從 CJS 切換到 ESM 將會是對兼容性的重大突破。

CJS 不能 require ESM

CJS 不能 require ESM 的最簡單原因是因為 ESM 可以做 top-level await，而 CJS scripts 不行。

top-level await 可以讓我們在 async function 之外使用 await 關鍵字。
ESM 的 multi-pasge loader 使 ESM 可以實現 top-level await。

由於 CJS 不支援 top-level await，因此甚至不可能將 ESM top-level await 轉換為 CJS。
